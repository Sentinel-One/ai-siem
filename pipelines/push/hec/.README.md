# pipelines/push/hec/

Pipelines for vendors that POST events **directly to an HEC endpoint** with
vendor-specific shaping (token handling, field unwrapping, batching, retry
semantics).

## Layout

```
push/hec/<vendor>/<product>/
  ├── metadata.yaml
  └── observo_export_pipeline_*.json
```

## Required metadata

```yaml
metadata_details:
  vendor: "<canonical_vendor_key>"
  product: "<canonical_product_key>"
  ingest_mode: "HEC"
  auth_type: "HEC Token | Bearer Token | API Key & Secret | Basic"
  # ... plus the standard pipeline fields
```
