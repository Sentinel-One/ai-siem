# transform_ocsf/

OCSF-compliant Lua serializers for Observo Pipeline Manager, ready to be
imported as `OCSFSerializer` transform nodes. 123 transforms covering the
most common SentinelOne AI SIEM sources.

## What's in each subdirectory

Each immediate subdirectory is one transform and contains:

| File | Purpose |
|---|---|
| `<transform_name>.json` | Observo-importable transform node definition. `template_name = OCSFSerializer`. Includes the inline Lua source, parameters, OCSF class mapping metadata, and the full harness validation record. |
| `metadata.yaml` | Human-readable description, same `metadata_details:` schema as the other `pipelines/community/<pipeline>/metadata.yaml` files. |
| `serializer.lua` | Standalone `.lua` (same content as the `lua_code` parameter in `<transform_name>.json`). |
| `sample.json` | Realistic source event generated by HELIOS/Jarvis for reference and testing (present when a matching generator exists). |

## Grades

Every transform exposes a harness grade as a top-level field in both
`<transform>.json` and `metadata.yaml`:

```json
{
  "grade": {
    "letter": "B",
    "score": 85,
    "verdict": "signed_off",
    "required_field_coverage_pct": 100,
    "source_field_coverage_pct": 100,
    "tested_with_realistic_event": true,
    "validated_at": "2026-04-19"
  }
}
```

See [GRADES.md](GRADES.md) for a one-table overview of every transform's
grade, independent AI review verdict, and OCSF class.

## Verdicts

| Verdict | Meaning |
|---|---|
| `signed_off` | Harness and independent AI review both pass. Safe to ship. |
| `analyzer_limit` | Script uses dynamic `class_uid` dispatch (mapping-table) that our static analyzer cannot resolve. The script is verified good in production; grade is artificially low. |
| `edge_case` | Multi-format combo script -- lower per-format coverage is structural, not a defect. |

## Validation

Every transform in this directory was graded by a 5-module harness:

1. `lupa` Lua runtime syntax check
2. 16-rule linter (processEvent contract, type guards, OCSF required fields)
3. OCSF `class_uid` resolution (5 patterns, including helper-call and mapping-table dispatch)
4. Source and target field coverage
5. Execution against a realistic source event with timestamp-type fuzzing

They were additionally cross-checked by independent AI review (library-level
pass plus per-outlier pass). 7 transforms that failed review are held back
for classification/mapping remediation and will ship in a follow-up PR.

## Installing a transform into an Observo pipeline

1. Open the destination pipeline in Observo Pipeline Manager.
2. Add a new node -> Transform -> OCSFSerializer.
3. In the transform dialog, import the contents of `<transform_name>.json`
   (the `lua_code` parameter carries the full serializer).
4. Wire the transform between the source collector and the SentinelOne
   destination, then save and deploy the pipeline.

## Contributing a new transform

Place a new subdirectory under `transform_ocsf/` with the four-file layout
above. Each `<transform_name>.json` must include a `grade` block so
reviewers can see the harness verdict at a glance.
